# spigot.yml 配置详解 - 概述

## 简介

spigot.yml 是 Spigot 服务端引入的核心配置文件，用于控制 Spigot 特有的功能和性能优化选项。所有基于 Spigot 开发的下游服务端（包括 Paper、Purpur、Pufferfish 等）都会继承并使用这个配置文件。

与 bukkit.yml 相比，spigot.yml 提供了更多针对性能优化的配置选项，包括实体激活范围、物品合并、区块加载等高级设置。这些配置对服务器性能有显著影响，合理调整可以大幅提升服务器的运行效率。

需要注意的是，如果你使用的是 Paper 或其他下游服务端，部分配置可能已被各自的配置文件（如 paper-world-defaults.yml）中的同名选项覆盖或扩展。

## 文件位置

spigot.yml 位于服务端根目录下，与 server.jar 同级：

```
服务端根目录/
├── server.jar
├── server.properties
├── bukkit.yml
├── spigot.yml          ← 本文件
├── plugins/
└── world/
```

服务端首次启动时会自动生成此文件。如果文件被删除，重启服务端后会重新生成默认配置。

## 配置文件结构

spigot.yml 主要包含以下几个部分：

| 节点 | 说明 | 详细文档 |
|------|------|----------|
| `settings` | 全局设置，包括调试、网络、玩家移动检测等 | [基础设置](基础设置.md)、[玩家设置](玩家设置.md) |
| `commands` | 命令相关配置 | [命令与消息](命令与消息.md) |
| `messages` | 服务器消息自定义 | [命令与消息](命令与消息.md) |
| `world-settings` | 世界级别的配置，包括实体、生成、性能等 | [世界设置](世界设置.md)、[性能优化](性能优化.md) |

## 配置修改注意事项

1. **备份配置**：修改前建议备份原配置文件
2. **重启生效**：大部分配置需要重启服务器才能生效
3. **语法检查**：YAML 格式对缩进敏感，建议使用专业编辑器
4. **逐步调整**：性能相关配置建议逐步调整，观察效果后再继续
