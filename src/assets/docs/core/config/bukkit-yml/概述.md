# bukkit.yml 配置详解 - 概述

## 简介

bukkit.yml 是 Bukkit 系服务端的核心配置文件之一。所有基于 CraftBukkit 开发的服务端（包括 Spigot、Paper、Purpur 等）都会使用这个配置文件。它主要控制服务器的基础行为设置、生物生成限制、区块垃圾回收以及各种刷新频率等参数。

与 server.properties 不同，bukkit.yml 使用 YAML 格式，提供了更多针对 Bukkit API 层面的配置选项。对于使用 Spigot 或 Paper 等下游服务端的用户，部分配置可能已被各自的配置文件（如 spigot.yml、paper.yml）中的同名选项覆盖或扩展。

## 文件位置

bukkit.yml 位于服务端根目录下，与 server.jar 同级：

```
服务端根目录/
├── server.jar
├── server.properties
├── bukkit.yml          ← 本文件
├── spigot.yml
├── plugins/
└── world/
```

服务端首次启动时会自动生成此文件。如果文件被删除，重启服务端后会重新生成默认配置。

## 配置文件结构

bukkit.yml 主要包含以下几个配置节点：

| 节点 | 说明 |
|------|------|
| `settings` | 服务器基础设置，如末地开关、权限文件、连接限制等 |
| `spawn-limits` | 各类生物的生成数量上限 |
| `chunk-gc` | 区块垃圾回收设置 |
| `ticks-per` | 各类生物的生成检查频率 |
| `autosave` | 自动保存间隔设置 |
| `aliases` | 命令别名配置 |
| `worlds` | 世界生成器和种子设置 |

## 配置文件格式

bukkit.yml 使用 YAML 格式，需要注意以下几点：

1. **缩进**：使用空格缩进，不要使用 Tab 键
2. **冒号后空格**：键值对的冒号后必须有空格
3. **字符串引号**：包含特殊字符的字符串需要用引号包裹
4. **大小写敏感**：配置键名区分大小写

示例：
```yaml
settings:
  allow-end: true
  shutdown-message: "服务器已关闭"
```

## 修改配置的注意事项

1. **备份**：修改前建议备份原配置文件
2. **重启生效**：大部分配置需要重启服务器才能生效
3. **语法检查**：修改后可使用在线 YAML 验证工具检查语法
4. **逐项修改**：建议每次只修改少量配置，便于排查问题
