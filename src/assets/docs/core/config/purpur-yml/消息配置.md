# Purpur.yml 消息配置

`messages` 部分允许你自定义服务器发送给玩家的各种消息文本。

---

## 基本消息

### 无法骑乘消息

```yaml
messages:
  cannot-ride-mob: "&c你没有权限骑乘这个生物"
```

| 配置项 | 说明 |
|--------|------|
| 类型 | 字符串 |
| 默认值 | `"&c你没有权限骑乘这个生物"` |
| 作用 | 当玩家尝试骑乘没有权限骑乘的生物时显示的消息 |

### AFK 相关消息

```yaml
messages:
  afk-broadcast-away: "&e&o%s 现在处于 AFK 状态"
  afk-broadcast-back: "&e&o%s 不再处于 AFK 状态"
  afk-tab-list-prefix: "[AFK] "
  afk-tab-list-suffix: ""
```

| 配置项 | 说明 |
|--------|------|
| afk-broadcast-away | 玩家进入 AFK 状态时的广播消息 |
| afk-broadcast-back | 玩家退出 AFK 状态时的广播消息 |
| afk-tab-list-prefix | AFK 玩家在 Tab 列表中的前缀 |
| afk-tab-list-suffix | AFK 玩家在 Tab 列表中的后缀 |

**占位符说明**：
- `%s` - 玩家名称

### 信标范围消息

```yaml
messages:
  demo-command-output: "已为 %s 播放演示屏幕"
```

| 配置项 | 说明 |
|--------|------|
| 类型 | 字符串 |
| 作用 | 执行演示命令时的输出消息 |

## 颜色代码

Purpur 消息支持 Minecraft 颜色代码：

| 代码 | 颜色 | 代码 | 颜色 |
|------|------|------|------|
| `&0` | 黑色 | `&8` | 深灰色 |
| `&1` | 深蓝色 | `&9` | 蓝色 |
| `&2` | 深绿色 | `&a` | 绿色 |
| `&3` | 深青色 | `&b` | 青色 |
| `&4` | 深红色 | `&c` | 红色 |
| `&5` | 紫色 | `&d` | 粉色 |
| `&6` | 金色 | `&e` | 黄色 |
| `&7` | 灰色 | `&f` | 白色 |

**格式代码**：

| 代码 | 效果 |
|------|------|
| `&l` | 粗体 |
| `&o` | 斜体 |
| `&n` | 下划线 |
| `&m` | 删除线 |
| `&k` | 随机字符 |
| `&r` | 重置格式 |

## 自定义消息示例

### 示例：自定义 AFK 消息

```yaml
messages:
  afk-broadcast-away: "&7[&e系统&7] &f%s &7已离开键盘"
  afk-broadcast-back: "&7[&e系统&7] &f%s &7已回到游戏"
  afk-tab-list-prefix: "&7[AFK] &f"
  afk-tab-list-suffix: ""
```

### 示例：禁用 AFK 广播

如果你不想广播 AFK 状态变化，可以将消息设置为空：

```yaml
messages:
  afk-broadcast-away: ""
  afk-broadcast-back: ""
```

## 权限相关消息

### 骑乘权限消息

```yaml
messages:
  cannot-ride-mob: "&c&l⚠ &c你没有权限骑乘这个生物！"
```

当玩家尝试骑乘需要权限的生物但没有相应权限时，会显示此消息。

**相关权限节点**：
- `purpur.ride.<mob_type>` - 允许骑乘特定类型的生物
- `purpur.ride.*` - 允许骑乘所有生物

---

## 注意事项

:::tip 提示
- 消息支持 MiniMessage 格式（如果服务器安装了相关支持）
- 空字符串 `""` 表示不发送该消息
- 修改消息后需要重启服务器或执行 `/purpur reload` 才能生效
:::

:::warning 注意
某些消息可能包含占位符（如 `%s`），请确保在自定义时保留这些占位符，否则可能导致错误。
:::
