# Purpur.yml 配置概述

## 什么是 Purpur？

Purpur 是基于 Paper 的分支服务端，在保持 Paper 所有功能的基础上，增加了大量自定义选项，让服务器管理员能够更精细地控制游戏行为。

`purpur.yml` 是 Purpur 服务端独有的配置文件，提供了许多 Paper 没有的功能选项。

## Purpur 的特色功能

### 🐾 生物行为自定义

Purpur 最大的特色是提供了丰富的生物行为自定义选项：

- **骑乘系统**：让玩家骑乘几乎任何生物（海豚、蜜蜂、末影龙等）
- **控制系统**：骑乘时可以控制生物移动方向
- **AI 修改**：自定义生物的 AI 行为
- **属性调整**：修改生物的生命值、移动速度等属性

### ⚡ 性能优化

- 减少不必要的网络数据包
- 优化实体处理
- 更高效的心跳检测机制

### 🎮 游戏机制调整

- 自定义附魔等级上限
- 修改经验获取规则
- 调整方块行为
- 更多合成配方选项

## 文件位置

```
服务端根目录/
├── purpur.yml          # Purpur 独有配置 ← 本文档说明的文件
├── paper.yml           # Paper 配置（旧版）
├── config/
│   └── paper-*.yml     # Paper 配置（新版）
├── spigot.yml
├── bukkit.yml
└── server.properties
```

首次启动 Purpur 服务端时会自动生成此文件。

## 配置文件结构

`purpur.yml` 主要分为以下几个部分：

| 节点 | 说明 | 详细文档 |
|------|------|----------|
| `settings` | 全局服务器设置 | [全局设置](./全局设置.md) |
| `messages` | 自定义消息文本 | [消息配置](./消息配置.md) |
| `world-settings` | 世界级别的配置 | [世界设置](./世界设置.md) |
| `mobs` | 生物行为配置 | [生物行为](./生物行为.md) |
| `blocks` | 方块相关配置 | [方块与物品](./方块与物品.md) |
| `gameplay` | 游戏机制配置 | [游戏机制](./游戏机制.md) |

## 配置修改建议

### 对于新手

1. **先备份**：修改前备份原始配置文件
2. **逐步修改**：每次只修改少量选项，测试后再继续
3. **查看日志**：修改后检查服务器日志是否有错误

### 对于进阶用户

1. **了解影响**：某些选项可能影响游戏平衡或性能
2. **测试环境**：建议先在测试服务器上验证配置
3. **版本兼容**：升级 Purpur 版本时注意配置文件变化

## 配置生效方式

大多数配置修改后需要重启服务器才能生效。部分配置支持热重载：

```
/purpur reload
```

:::warning 注意
热重载可能不会应用所有更改，建议重要配置修改后完全重启服务器。
:::

## 与其他配置文件的关系

Purpur 继承了 Paper 的所有配置，因此：

- `paper-global.yml` 和 `paper-world-defaults.yml` 中的配置仍然有效
- 如果 Purpur 和 Paper 有相同功能的配置，Purpur 的配置优先
- Spigot 和 Bukkit 的配置也会被继承
