# server.properties 概述

## 简介

`server.properties` 是 Minecraft Java 版服务端的核心配置文件。无论使用哪种服务端核心（原版、Spigot、Paper、Forge 等），服务端首次启动时都会自动生成此文件。通过修改该文件中的配置项，服务器管理员可以控制服务器的基本行为、网络设置、世界生成、游戏规则等方面。

## 文件位置

`server.properties` 位于服务端根目录下，与服务端 JAR 文件同级。

```
服务端根目录/
├── server.jar
├── server.properties    ← 配置文件
├── eula.txt
├── logs/
└── world/
```

## 配置格式说明

- 每行一个配置项，格式为 `键=值`
- 以 `#` 开头的行为注释
- 修改配置后需要重启服务器才能生效
- 布尔值使用 `true` 或 `false`
- 字符串值直接填写，无需引号

## 编辑方式

### 使用文本编辑器

1. 停止服务器（推荐，避免配置被覆盖）
2. 使用任意文本编辑器打开 `server.properties`
3. 修改需要的配置项
4. 保存文件
5. 重新启动服务器

### 常用文本编辑器推荐

- **Windows**: 记事本、Notepad++、VS Code
- **Linux**: nano、vim、VS Code
- **macOS**: TextEdit、VS Code、Sublime Text

### 注意事项

- 修改前建议备份原文件
- 确保文件编码为 UTF-8（特别是包含中文时）
- 不要在值的两端添加多余的空格
- 布尔值必须是小写的 `true` 或 `false`

## 快速参考

以下是一些最常用的配置项：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `server-port` | `25565` | 服务器端口 |
| `max-players` | `20` | 最大玩家数 |
| `online-mode` | `true` | 正版验证 |
| `difficulty` | `easy` | 游戏难度 |
| `gamemode` | `survival` | 默认游戏模式 |
| `motd` | `A Minecraft Server` | 服务器描述 |
| `white-list` | `false` | 白名单开关 |
| `pvp` | `true` | PVP 开关 |
