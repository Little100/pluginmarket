# Minecraft 服务端选择指南

## 概述

Minecraft 服务端根据功能特性可分为多个类别：纯插件端、纯模组端、混合端、代理端和跨平台端。每种类型都有其特定的使用场景和优缺点。选择合适的服务端核心是搭建服务器的第一步，也是影响服务器性能、稳定性和可扩展性的关键因素。

## 服务端分类

### 纯插件端

纯插件端基于 Bukkit API，支持通过插件扩展服务器功能，不支持 Forge/Fabric 模组。

| 服务端 | 简介 |
|--------|------|
| **Spigot** | Bukkit 的优化分支，插件生态的基础 |
| **Paper** | Spigot 的优化分支，性能更好，修复更多漏洞 |
| **Purpur** | Paper 的分支，提供更多自定义配置选项 |
| **Pufferfish** | Paper 的高性能分支，专注于大型服务器优化 |
| **Folia** | Paper 的多线程分支，适合高并发场景 |
| **Leaves** | Paper 的分支，专注于原版特性保留，适合技术向服务器 |

### 纯模组端

纯模组端支持加载模组（Mod），通过模组实现内容扩展，不支持 Bukkit 插件。

| 服务端 | 简介 |
|--------|------|
| **Forge** | 历史最悠久的模组加载器，模组生态最丰富 |
| **NeoForge** | Forge 的社区分支，1.20.1+ 版本的新选择 |
| **Fabric** | 轻量级模组加载器，启动快，更新及时 |
| **Quilt** | Fabric 的分支，兼容 Fabric 模组，提供更多功能 |

### 混合端（插件+模组）

混合端同时支持 Bukkit 插件和 Forge/Fabric 模组，但稳定性和兼容性通常不如纯净端。

| 服务端 | 简介 | 状态 |
|--------|------|------|
| **Arclight** | 基于 Forge/Fabric/NeoForge 的混合端，目前最活跃 | ✅ 活跃维护 |
| **CatServer** | 基于 Forge 的混合端，国内团队开发 | ✅ 活跃维护 |
| **Mohist** | 基于 Forge 的混合端 | ⚠️ 已停止更新 |
| **Banner** | 基于 Fabric 的混合端 | ⚠️ 已停止更新 |

### 代理端

代理端用于连接多个子服务器，实现群组服功能。

| 服务端 | 简介 | 状态 |
|--------|------|------|
| **BungeeCord** | 最早的代理端，Spigot 团队开发 | ⚠️ 维护中但不推荐新项目使用 |
| **Waterfall** | BungeeCord 的优化分支，Paper 团队开发 | ❌ 已 EOL（End of Life） |
| **Velocity** | 现代化代理端，性能和安全性更好 | ✅ 推荐使用 |

### 跨平台端

| 服务端 | 简介 |
|--------|------|
| **Geyser** | 允许基岩版玩家加入 Java 版服务器的桥接工具 |

## 按场景推荐

### 纯插件服（生存/小游戏/RPG）

**推荐：Paper 或 Purpur**

- Paper 是目前最主流的插件端，性能优秀，插件兼容性极高
- Purpur 在 Paper 基础上提供更多配置选项，适合需要深度自定义的服务器
- 两者的插件兼容性几乎相同，可以无缝切换

### 纯模组服

**大型整合包 → Forge**
- 模组生态最丰富，大多数知名整合包基于 Forge
- 适合工业、魔法、冒险等大型模组

**轻量模组/原版增强 → Fabric**
- 启动速度快，内存占用低
- 更新速度快，通常在新版本发布后很快适配
- 适合优化模组、小型内容模组

### 插件+模组混合服

**推荐：Arclight**

- 目前最活跃的混合端项目
- 支持 Forge、Fabric、NeoForge 多种模组加载器
- 版本覆盖范围广

### 高并发/大型服务器

**多区块活动场景 → Folia**
- Paper 团队开发的多线程服务端
- 适合有大量玩家分布在不同区块的服务器
- 注意：大多数插件需要专门适配 Folia

**通用高性能场景 → Pufferfish**
- 在 Paper 基础上进行深度优化
- 插件兼容性比 Folia 更好
- 适合高玩家数但不需要多线程的服务器

### 技术向/红石服务器

**推荐：Leaves**

- 专注于保留原版特性
- 修复了 Paper 中一些影响红石和技术玩法的改动
- 适合生电服、红石服、技术生存服

### 群组服/多服互联

**推荐：Velocity**

- Paper 团队开发的现代化代理端
- 性能优于 BungeeCord
- 安全性更高，支持现代转发模式
- 插件生态完善

### 基岩版+Java版互通

**推荐：Geyser**

- 可作为插件或独立代理运行
- 允许基岩版玩家（手机、主机、Windows 10/11）加入 Java 版服务器
- 通常配合 Floodgate 使用，实现免 Java 账号登录

### NeoForge vs Forge 的选择

| 场景 | 推荐 |
|------|------|
| 1.20.1 及更早版本 | Forge |
| 1.20.2 及更新版本 | NeoForge |
| 使用的模组只支持 Forge | Forge |
| 使用的模组只支持 NeoForge | NeoForge |
| 新开服且模组都支持 | NeoForge（更活跃的开发） |

NeoForge 是 Forge 社区在 2023 年分裂后产生的分支。对于 1.20.1 及更早版本，Forge 仍是主流选择；对于更新版本，NeoForge 的开发更加活跃，建议优先考虑。

## 版本选择建议

### 最新版 vs 长期支持版

| 选择 | 适用场景 |
|------|----------|
| 最新版 | 想体验新内容、插件/模组生态已跟进 |
| 长期支持版（如 1.20.1、1.21.1） | 追求稳定、插件/模组生态成熟 |

建议：
- 纯插件服可以较快跟进新版本
- 模组服建议等待模组生态稳定后再升级

### 1.8 PVP 服的选择

1.8 版本因其独特的 PVP 机制仍有大量玩家群体。

**推荐：Paper 1.8.8**
- 仍有维护的 1.8 分支
- 插件兼容性好

**注意事项：**
- 1.8 版本的安全漏洞较多，需要做好防护
- 部分现代插件不再支持 1.8

### 模组服的版本选择

| 版本 | 特点 | 推荐场景 |
|------|------|----------|
| 1.12.2 | 模组数量最多，经典版本 | 工业/魔法整合包 |
| 1.16.5 | 下界更新，模组生态成熟 | 综合整合包 |
| 1.18.2 | 世界生成改版，洞穴更新 | 探索向整合包 |
| 1.19.2 | 荒野更新，生态较好 | 通用整合包 |
| 1.20.1 | 当前最新稳定版本 | 新整合包首选 |

**选择建议：**
- 如果有特定想玩的整合包，跟随整合包版本
- 新开模组服推荐 1.20.1（生态最活跃）
- 想玩经典老模组选择 1.12.2

## 注意事项

### 关于已停更的服务端

**Mohist**
- 曾是流行的 Forge 混合端
- 已于 2024 年停止更新
- 现有服务器建议迁移至 Arclight 或 CatServer

**Banner**
- 曾是主要的 Fabric 混合端
- 已停止更新
- 建议迁移至 Arclight（Fabric 版本）

### 关于 Waterfall EOL

Waterfall 已于 2024 年宣布 End of Life（生命周期结束）：
- 不再接收功能更新和安全修复
- 现有项目应尽快迁移至 Velocity
- 新项目不应使用 Waterfall

### 混合端的局限性

使用混合端前需了解以下限制：

1. **兼容性问题**
   - 并非所有插件都能与模组完美兼容
   - 部分模组可能与插件系统冲突
   - 需要花费更多时间调试

2. **稳定性风险**
   - 混合端的稳定性通常不如纯净端
   - 可能出现难以排查的崩溃问题
   - 更新频率可能不如上游项目

3. **性能开销**
   - 同时运行插件和模组系统会增加资源消耗
   - 需要更高的服务器配置

4. **技术支持**
   - 遇到问题时，插件/模组作者可能不提供混合端支持
   - 社区资源相对较少

**建议：** 如果可以通过纯插件或纯模组实现需求，优先选择纯净端。只有在确实需要同时使用插件和模组时，才考虑混合端。
