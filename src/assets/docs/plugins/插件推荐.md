## 插件安装与管理基础

在开始使用插件之前，了解插件的基本安装和管理方法非常重要。

### 如何安装插件

1. **下载插件文件**：从可信的插件平台下载 `.jar` 格式的插件文件
2. **放入 plugins 目录**：将下载的 `.jar` 文件放入服务器根目录下的 `plugins` 文件夹
3. **重启服务器**：完全关闭服务器后重新启动，插件将自动加载
4. **验证安装**：使用 `/plugins` 或 `/pl` 命令查看已加载的插件列表

> **注意**：虽然部分插件支持热重载（`/reload`），但这可能导致内存泄漏或数据异常。建议始终通过完全重启服务器来加载新插件。

### 插件配置文件基础

插件首次加载后，会在 `plugins/插件名称/` 目录下生成配置文件：

- **config.yml**：主配置文件，包含插件的核心设置
- **messages.yml**：消息配置文件，用于自定义插件输出的文本
- **data.yml** 或数据库文件：存储插件运行时产生的数据

配置文件使用 YAML 格式，修改时需注意：
- 保持正确的缩进（使用空格，不要使用 Tab）
- 字符串值如包含特殊字符需用引号包裹
- 修改后需重启服务器或使用插件提供的重载命令

### 插件冲突排查

当服务器出现异常时，可能是插件冲突导致的。排查步骤：

1. **查看控制台日志**：启动时的错误信息通常会指明问题插件
2. **二分法排查**：移除一半插件，逐步缩小问题范围
3. **检查依赖关系**：确保所有前置插件已正确安装
4. **版本兼容性**：确认插件版本与服务端版本匹配

## 基础管理类

基础管理类插件是服务器运营的核心，提供玩家管理、权限控制等基本功能。

### EssentialsX

**简介**：EssentialsX 是 Bukkit 系服务器最经典的基础命令插件，提供超过 100 个实用命令，涵盖传送、经济、玩家管理等方方面面。它是原 Essentials 插件的社区维护版本，持续更新并支持最新版本的 Minecraft。

**主要功能**：
- 基础命令：`/gamemode`、`/time`、`/weather`、`/heal`、`/feed` 等
- 传送系统：`/home`、`/warp`、`/tpa`、`/spawn` 等
- 玩家管理：`/kick`、`/ban`、`/mute`、`/jail` 等
- 内置经济系统（需配合 Vault 使用）
- 聊天格式化和昵称系统

**适用平台**：Spigot、Paper、Purpur 及其分支

**下载地址**：
- SpigotMC: https://www.spigotmc.org/resources/essentialsx.9089/
- Modrinth: https://modrinth.com/plugin/essentialsx
- GitHub: https://github.com/EssentialsX/Essentials

### Vault

**简介**：Vault 是一个 API 桥接插件，为其他插件提供统一的经济、权限和聊天接口。它本身不提供任何功能，而是作为中间层让不同的经济插件和权限插件能够相互配合工作。几乎所有涉及经济或权限的插件都依赖 Vault。

**主要功能**：
- 经济 API：统一的货币操作接口
- 权限 API：统一的权限检查接口
- 聊天 API：统一的前缀/后缀获取接口

**适用平台**：Bukkit、Spigot、Paper 及其分支

**下载地址**：
- SpigotMC: https://www.spigotmc.org/resources/vault.34315/
- GitHub: https://github.com/milkbowl/Vault

> **重要**：Vault 是许多插件的前置依赖，建议所有服务器都安装此插件。

- 自定义命令和别名
- 计划任务系统

**适用平台**：Spigot、Paper 及其分支

**下载地址**：
- SpigotMC: https://www.spigotmc.org/resources/cmi.3742/（付费插件）

## 跨服相关

跨服插件用于管理多服务器网络，实现数据同步和玩家无缝切换。

### 代理端简介

跨服网络需要使用代理端（Proxy）来管理多个后端服务器：

- **Velocity**：现代化的代理端，性能优秀，安全性高，推荐使用
- **BungeeCord**：经典的代理端，生态成熟，插件丰富

> 关于代理端的详细介绍，请参阅服务端核心文档中的 [Velocity](../core/velocity.md) 和 [BungeeCord](../core/bungeecord.md) 章节。

### LuckPerms（跨服权限同步）

**简介**：LuckPerms 不仅是单服权限插件，还支持跨服权限同步。通过配置数据库存储，可以在多个服务器间共享权限数据。

**跨服功能**：
- MySQL/MariaDB/PostgreSQL 数据库存储
- 实时权限同步
- 服务器特定权限（使用上下文）
- 网络范围的权限管理

**配置要点**：
1. 所有服务器使用相同的数据库
2. 配置 `server` 选项区分不同服务器
3. 使用上下文设置服务器特定权限

**下载地址**：参见基础管理类中的 LuckPerms 条目

## 总结

本文档介绍了 Bukkit/Spigot/Paper 系服务器常用的各类插件。在选择插件时，请注意以下几点：

1. **按需选择**：不要安装过多不必要的插件，每个插件都会占用服务器资源
2. **版本兼容**：确保插件版本与服务端版本兼容
3. **来源可靠**：只从官方或可信平台下载插件
4. **定期更新**：保持插件更新以获得最新功能和安全修复
5. **备份数据**：安装新插件前备份服务器数据

如果你是新手，建议从以下基础插件组合开始：

- **EssentialsX**：基础命令
- **LuckPerms**：权限管理
- **Vault**：API 桥接
- **WorldGuard + WorldEdit**：区域保护
- **CoreProtect**：方块记录

随着服务器发展，再根据需求逐步添加其他插件。
