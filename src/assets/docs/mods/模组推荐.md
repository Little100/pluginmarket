# 模组推荐

本指南面向 Minecraft 服务端管理员，介绍适用于 Forge、NeoForge 和 Fabric 服务端的常用模组。

---

## 一、模组安装基础

### 1.1 服务端模组 vs 客户端模组 vs 双端模组

| 类型 | 说明 | 示例 |
|------|------|------|
| **服务端模组** | 仅需安装在服务端，客户端无需安装 | Lithium、ServerCore、Spark |
| **客户端模组** | 仅需安装在客户端，用于改善玩家体验 | OptiFine、Sodium、小地图 |
| **双端模组** | 服务端和客户端都需要安装 | WorldEdit、大多数内容模组 |

**判断方法：**
- 查看模组页面说明，通常会标注 "Server-side only" 或 "Client-side only"
- 如果模组添加了新方块、物品或实体，通常是双端模组
- 纯优化类模组大多是单端模组

### 1.2 如何安装服务端模组

1. **确认服务端类型**：确保服务端是 Forge、NeoForge 或 Fabric
2. **下载对应版本的模组**：版本必须与服务端的 Minecraft 版本和加载器版本匹配
3. **放入 mods 文件夹**：将 `.jar` 文件放入服务端根目录下的 `mods` 文件夹
4. **重启服务端**：模组会在启动时自动加载
5. **检查日志**：查看启动日志确认模组是否正常加载

**文件结构示例：**
```
server/
├── mods/
│   ├── lithium-fabric-0.12.1.jar
│   ├── ferritecore-6.0.0-fabric.jar
│   └── spark-1.10.53-fabric.jar
├── config/
├── world/
└── fabric-server-launch.jar
```

### 1.3 模组版本兼容性注意事项

- **Minecraft 版本**：模组必须与服务端的 Minecraft 版本完全匹配
- **加载器版本**：Forge 模组不能用于 Fabric，反之亦然
- **依赖模组**：部分模组需要前置库（如 Fabric API），必须一并安装

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 服务端无法启动 | 模组版本不匹配 | 检查所有模组的目标版本 |
| 缺少依赖 | 未安装前置模组 | 查看模组页面的依赖列表 |
| 模组冲突 | 两个模组修改了相同功能 | 移除其中一个或查找兼容补丁 |

### 1.4 推荐的模组下载平台

| 平台 | 网址 | 特点 |
|------|------|------|
| **CurseForge** | https://www.curseforge.com/minecraft/mc-mods | 模组数量最多，历史悠久 |
| **Modrinth** | https://modrinth.com/mods | 界面现代，审核严格，下载速度快 |
| **GitHub** | 各模组仓库 | 获取开发版本或源码 |

Sinytra Connector 允许在 Forge/NeoForge 服务端上运行部分 Fabric 模组。

| 属性 | 信息 |
|------|------|
| 适用加载器 | Forge / NeoForge |

| 前置模组 | Forgified Fabric API |

**下载地址：**
- CurseForge: https://www.curseforge.com/minecraft/mc-mods/sinytra-connector
- Modrinth: https://modrinth.com/mod/connector
- GitHub: https://github.com/Sinytra/Connector

**注意事项：**
- 并非所有 Fabric 模组都能正常运行
- 可能存在兼容性问题
- 建议在测试环境验证

### 7.2 混合端注意事项

混合端（如 Mohist、Arclight）允许同时运行模组和插件，但存在以下问题：

| 问题 | 说明 |
|------|------|
| 兼容性问题 | 部分模组或插件可能无法正常工作 |
| 性能损耗 | 兼容层会带来额外的性能开销 |
| 更新滞后 | 混合端通常比官方加载器更新慢 |
| 调试困难 | 问题排查更加复杂 |

**建议：**
- 如果可能，优先选择纯模组端或纯插件端
- 使用混合端前充分测试所有模组和插件
- 关注混合端的官方兼容性列表

## 八、模组服开服注意事项

### 8.1 客户端模组同步问题

- 双端模组必须在服务端和客户端都安装相同版本
- 服务端模组不需要客户端安装
- 可以使用模组包分发工具（如 CurseForge App、Prism Launcher）简化客户端配置

### 8.2 模组冲突排查方法

1. **二分法**：移除一半模组，测试是否正常，逐步缩小范围
2. **查看日志**：启动日志和崩溃报告通常会指出问题模组
3. **检查已知冲突**：查看模组页面的已知问题和不兼容列表
4. **使用 Spark**：分析性能问题时使用 Spark 生成报告

### 8.3 内存需求估算

| 模组数量 | 建议内存 |
|----------|----------|
| 10-30 个 | 4-6 GB |
| 30-100 个 | 6-10 GB |
| 100+ 个 | 10-16 GB |

**注意：**
- 以上为服务端内存建议
- 实际需求取决于具体模组和玩家数量
- 建议预留 20% 的内存余量

### 8.4 模组更新策略

- **稳定优先**：生产服务器不要频繁更新模组
- **测试环境**：在测试服务器验证更新后再应用到生产环境
- **备份**：更新前务必备份世界和配置文件
- **版本锁定**：记录当前使用的模组版本，便于回滚
- **关注更新日志**：了解更新内容，特别是破坏性变更
